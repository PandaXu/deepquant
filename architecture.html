🏗️ 系统架构总览

VeighNa 采用分层事件驱动架构。底层事件引擎驱动整个系统，Gateway 层负责对接各交易所/券商，MainEngine 协调所有功能引擎，App 层提供可插拔的业务模块。

📁 模块详解

7 大功能模块，各司其职

⚡ Event 事件引擎

vnpy/event/
  • EventEngine — 事件驱动核心，双线程运行（事件分发 + 定时器）
  • Event — 事件对象 (type + data)
  • 支持按类型注册 handler + 通用 handler
  • 每秒产生 eTimer 定时事件
  • 线程安全的 Queue 机制

💹 Trader 交易核心

vnpy/trader/
  • MainEngine — 平台主引擎，管理 gateways/engines/apps
  • OmsEngine — 订单管理，缓存 tick/order/trade/position 等数据
  • BaseGateway — 交易接口抽象，连接交易所
  • BaseDatafeed — 数据服务抽象
  • BaseDatabase — 数据库抽象（默认 SQLite）
  • OffsetConverter — 期货平仓转换（平今/平昨/锁仓）
  • BarGenerator — K 线合成（Tick→1分钟→N分钟→小时→日线）
  • ArrayManager — 技术指标计算（50+ TA-Lib 指标）
  • OptimizationSetting — 参数优化（穷举/遗传算法）
  • 10 种 Request/Data 数据对象
  • 38 个交易所枚举（CFFEX, NYSE, NASDAQ...）

🔬 Alpha 研究平台

vnpy/alpha/
  • AlphaLab — 研究实验室，数据存储（Parquet/Shelve/Pickle）
  • AlphaDataset — 数据集模板（特征表达式 + 预处理器）
  • AlphaModel — ML 模型基类（fit/predict）
  • AlphaStrategy — 策略模板（on_bars + 目标仓位执行）
  • BacktestingEngine — 多品种回测引擎
  • 内建 Alpha101 / Alpha158 因子集
  • 截面/时序/技术分析/数学函数
  • LightGBM / MLP / Lasso 模型实现

🌐 RPC 分布式

vnpy/rpc/
  • RpcServer — ZeroMQ REP + PUB 模式服务端
  • RpcClient — ZeroMQ REQ + SUB 模式客户端
  • 支持远程函数注册与调用
  • 心跳检测机制（10s 间隔 / 30s 容错）
  • 发布-订阅模式数据推送

📊 Chart 可视化

vnpy/chart/
  • ChartWidget — K 线图组件
  • ChartItem — 图表元素（K线/成交量/指标）
  • ChartAxis — 坐标轴系统
  • ChartManager — 图表管理器

🖥️ UI 桌面界面

vnpy/trader/ui/
  • MainWindow — Qt 主窗口
  • QtWidgets — 交易组件（下单/持仓/日志面板）
  • PySide6/PyQt5 兼容层
  • 菜单式 App 管理

📝 Examples 示例

examples/
  • veighna_trader — 完整交易平台启动
  • client_server — RPC 服务/客户端示例
  • data_recorder — 行情录制
  • candle_chart — K 线图表
  • no_ui — 无界面运行
  • simple_rpc — 简单 RPC 测试
