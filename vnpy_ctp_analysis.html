📦 第 1 层：CTP 原生 SDK

  • 平台：Windows (.dll) / Linux (.so) / macOS (.framework unified binary x86_64+arm64)
  • 行情 API：thostmduserapi_se — 创建连接、订阅行情、接收行情推送
  • 交易 API：thosttraderapi_se — 登录认证、下单撤单、查询资金持仓、接收成交推送
  • 通信协议：TCP 长连接，CTP 自定义二进制协议
  • SDK 版本差异：macOS SDK 是较旧版本，缺少 ReqQryUserSession、ReqOffsetSetting 等新 API

🔗 第 2 层：C++ pybind11 绑定（vnpy_ctp/api/vnctp/）

  • vnctpmd.cpp/h — MdApi：行情接口绑定，管理 TCP 连接、订阅/退订、接收行情推送
  • vnctptd.cpp/h — TdApi：交易接口绑定，管理登录认证、下单/撤单、查询、接收成交/委托推送
  • vnctp.h：公共工具 — getString/getInt/getChar/getDouble 等函数从 Python dict 提取字段并填充 C struct
  • Task 队列模型：所有 CTP 回调在独立线程中触发，通过 Task 队列将数据安全传递到 Python 线程
  • 每个 CTP API 函数/回调都被封装为 pybind11 方法，输入输出统一使用 py::dict

🐍 第 3 层：Python Gateway（ctp_gateway.py）

  • CtpGateway：继承 BaseGateway，持有 CtpMdApi + CtpTdApi 实例，实现 connect/subscribe/send_order/cancel_order/close
  • CtpMdApi：继承 C++ MdApi，重写 onRtnDepthMarketData 等回调，将 CTP 行情 dict 转换为 VeighNa TickData 并通过 gateway.on_tick() 推送
  • CtpTdApi：继承 C++ TdApi，重写 onRtnOrder/onRtnTrade/onRspQryInvestorPosition 等回调，转换并推送 OrderData/TradeData/PositionData/AccountData/ContractData
  • 常量映射表：11 个字典实现 CTP 字符常量 ↔ VeighNa Enum 双向转换

⚙️ C++ 绑定层详解

核心设计思想：用 pybind11 将 CTP C++ SDK 封装为 Python 可调用的类，所有数据交换统一使用 py::dict

2.1 Task 队列线程模型

sequenceDiagram participant CTP as CTP 服务器 participant SDK as C++ SDK
(独立线程) participant Task as Task 队列 participant GIL as Python GIL
主线程 participant Py as Python 回调 CTP->>SDK: 行情/成交数据推送 SDK->>Task: new Task(data)
加入队列 Note over Task: 线程安全队列 Task-->>GIL: py::gil_scoped_acquire GIL->>Task: 取出 Task Task->>Py: processTask(task) Py->>Py: onRtnDepthMarketData(dict) Py->>Py: gateway.on_tick(TickData)
关键设计：CTP SDK 回调在独立网络线程触发，不能直接调用 Python（无 GIL）。通过 Task 队列 + py::gil_scoped_acquire 将数据安全传递到 Python 主线程。MdApi 和 TdApi 各有独立的 Task 队列和处理循环。

vnctp.h 公共工具

// 从 Python dict 安全提取字段并填充 C struct 的辅助函数
void getString(const py::dict &req, const char *key, char *field, int size);
void getInt(const py::dict &req, const char *key, int *field);
void getChar(const py::dict &req, const char *key, char *field);
void getDouble(const py::dict &req, const char *key, double *field);

// 创建 C struct 时调用 memset 清零，避免脏数据
// 零值字段自动跳过，不写入返回 dict（节省带宽）
设计要点： ① 双向都用 py::dict 作为通用数据载体，避免为每个 CTP 结构体写独立 pybind11 绑定
② 零值字段跳过：CTP 结构体字段极多（某些超过 120 个），但实际使用很少，返回 dict 中只包含非零字段以减少序列化开销
③ GBK 编码：CTP 使用 GBK 编码中文字段，绑定层自动在 GBK ↔ UTF-8 间转换

3.2 CtpMdApi — 行情处理

核心回调：onRtnDepthMarketData(data: dict)

处理流程：
① 过滤无时间戳的异常数据 → ② 从 symbol_contract_map 获取合约信息（未就绪则丢弃）
③ 处理大商所(DCE)特殊逻辑：忽略 ActionDay，用本地日期
④ 拼接时间戳 → ⑤ 构造 TickData（含 5 档盘口）→ ⑥ gateway.on_tick(tick) 推入事件引擎

3.3 CtpTdApi — 交易处理

委托下单 send_order(req: OrderRequest) → str (vt_orderid)

处理流程：
① 校验开平方向/委托类型是否支持 → ② order_ref 自增作为本地委托编号
③ 查映射表将 VeighNa 常量转为 CTP 字符常量
④ 构造 CTP 下单请求 dict（含 18 个字段）→ ⑤ reqOrderInsert(ctp_req, reqid)
⑥ 立即创建 OrderData(SUBMITTING) 通过 on_order() 推送（本地预确认）
⑦ 订单 ID 规则：{FrontID}_{SessionID}_{OrderRef}，全局唯一
委托推送 onRtnOrder(data: dict)

① 合约信息未就绪 → 缓存到 self.order_data 列表，等合约就绪后重放
② 从 CTP 状态码映射 VeighNa Status → ③ 处理特殊拒单（提交被拒 → REJECTED）
④ 解析时间戳/委托类型 → ⑤ 构造 OrderData → ⑥ gateway.on_order(order)
持仓查询 onRspQryInvestorPosition — 最复杂的回调

① CTP 分品种多次回调，每个品种一个 PositionData
② 上期所 SHFE/INE 特殊处理：直接用 YdPosition 作为昨仓
③ 其他交易所：昨仓 = 总持仓 - 今仓
④ 累加多笔查询结果（同一品种可能多次返回）
⑤ 加权平均计算持仓成本：cost += PositionCost; price = cost / (volume * size)
⑥ last=true 时统一推送所有持仓并清空缓存

🗺️ 常量映射体系

CTP 使用单字符编码表示状态/方向/类型，VeighNa 使用 Python Enum。11 个双向映射字典桥接两者。

映射表方向示例
STATUS_CTP2VT CTP → VeighNa '0' → Status.ALLTRADED   '3' → Status.NOTTRADED   '5' → Status.CANCELLED
DIRECTION_VT2CTP VeighNa → CTP Direction.LONG → '0' (买)   Direction.SHORT → '1' (卖)
DIRECTION_CTP2VT CTP → VeighNa '0' → LONG   '1' → SHORT   '2' → LONG   '3' → SHORT
ORDERTYPE_VT2CTP VeighNa → CTP LIMIT → ('2', '3', '1') 限价+当日有效+任意数量   FAK → ('2', '1', '1') 限价+立即成交+任意数量
ORDERTYPE_CTP2VT CTP → VeighNa 三元组 (PriceType, TimeCondition, VolumeCondition) → OrderType
OFFSET_VT2CTP VeighNa → CTP OPEN → '0'   CLOSE → '1'   CLOSETODAY → '3'   CLOSEYESTERDAY → '4'
OFFSET_CTP2VT CTP → VeighNa 反向映射
EXCHANGE_CTP2VT CTP → VeighNa "CFFEX" → Exchange.CFFEX   "SHFE" → Exchange.SHFE ... (6 家期货交易所)
PRODUCT_CTP2VT CTP → VeighNa '1' → FUTURES   '2' → OPTION   '3' → SPREAD
OPTIONTYPE_CTP2VT CTP → VeighNa '1' → CALL   '2' → PUT
symbol_contract_map 全局缓存 {"rb2510" → ContractData, "IF2506" → ContractData, ...} 行情/交易回调必查
⚠️ 特殊处理：
① 大商所 DCE：行情推送中 ActionDay 字段为空或异常，需用本地日期替代
② 上期所 SHFE/INE：今仓/昨仓分离，OffsetConverter 需处理平今/平昨逻辑
③ 郑商所 CZCE：期权产品名称带 C/P 后缀，需截取处理
④ 浮点数极值：CTP 用 sys.float_info.max 表示无效价格，adjust_price() 将其归零
